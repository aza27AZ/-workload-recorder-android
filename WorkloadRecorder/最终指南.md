# 📱 工作量记录 - APK生成指南

## 🎯 目标：5分钟内获得APK

## 🚀 最简单的方法 - GitHub Actions

### 步骤1：准备项目（1分钟）

你的项目已经准备好了！在文件夹中：
```
d:\下载缓存\移动端工作量记录\WorkloadRecorder\
```

### 步骤2：创建GitHub仓库（2分钟）

1. 访问 https://github.com/new
2. 仓库名：`workload-recorder`
3. **不要勾选** "Add a README file"
4. **不要勾选** "Add .gitignore"
5. **不要勾选** "Choose a license"
6. 点击 "Create repository"

### 步骤3：上传代码（1分钟）

打开命令提示符（CMD）或PowerShell，执行：

```bash
cd d:\下载缓存\移动端工作量记录\WorkloadRecorder

git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/<你的用户名>/workload-recorder.git
git push -u origin main
```

⚠️ **注意**：将 `<你的用户名>` 替换为你的GitHub用户名

### 步骤4：自动构建并下载（1分钟）

1. 访问你的 GitHub 仓库页面
2. 点击顶部的 **Actions** 标签
3. 你会看到 "Build APK" workflow 正在运行
4. 等待 2-5 分钟，直到显示绿色的 ✅
5. 点击该 workflow run
6. 滚动到页面底部 **Artifacts** 区域
7. 点击 `workload-app-debug` 下载
8. 解压 ZIP 文件，得到 `app-debug.apk`

## ✅ 完成！

现在你有了APK文件，可以安装到Android设备上了！

---

## 📱 安装APK到手机

1. 通过微信/QQ/数据线将 `app-debug.apk` 发送到手机
2. 在手机上找到文件并点击
3. 如果提示"允许安装未知来源应用"，点击设置并允许
4. 点击"安装"
5. 安装完成后点击"打开"

## 🔧 如果遇到问题

### 问题1：GitHub Actions 没有自动运行？

**解决**：
1. 确认代码已推送到 `main` 分支
2. 确认 `.github/workflows/build.yml` 文件存在
3. 在 Actions 标签页点击 "Run workflow" 手动触发

### 问题2：构建失败了？

**解决**：
1. 点击失败的 workflow run
2. 查看详细的错误日志
3. 常见问题是网络问题导致依赖下载失败
4. 点击 "Re-run jobs" 重新构建

### 问题3：Gradle wrapper没有执行权限？

**解决**：
工作流中已经包含了 `chmod +x gradlew` 命令，应该没问题。
