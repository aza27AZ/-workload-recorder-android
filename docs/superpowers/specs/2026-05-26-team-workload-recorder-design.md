# 团队工作量记录 App - 设计文档

## 概述

一款安卓手机应用，用于团队管理员记录和统计团队成员的工作量，涵盖值班信息、授课信息、运营概况及外部预约管理。采用原生 Android 开发（Kotlin + Jetpack Compose + Room），本地存储为主，支持云备份和 Excel/文字导出。

---

## 一、技术选型

| 维度 | 选择 |
|------|------|
| 开发语言 | Kotlin |
| UI 框架 | Jetpack Compose（声明式 UI） |
| 本地数据库 | Room（Android 官方 SQLite ORM） |
| 架构模式 | MVVM（Model-View-ViewModel） |
| 云备份 | Android Auto Backup（系统级自动备份）+ 手动备份 |
| 导出 | Excel（Apache POI）/ 固定模板文字总结 |
| 构建工具 | Gradle + Android Studio |

## 二、数据模型

### 2.1 工作人员（Staff）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Long (PK, 自增) | 主键 |
| name | String | 姓名，必填 |
| remark | String | 备注，"值班人员" 或 "顶班人员"，默认"值班人员" |
| sortOrder | Int | 排序号，控制列表中显示顺序 |

### 2.2 值班记录（DutyRecord）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Long (PK, 自增) | 主键 |
| staffId | Long (FK → Staff) | 关联的工作人员 |
| dutyDate | LocalDate | 值班日期 |
| shiftType | Enum | 班次：早班 / 晚班 / 通班 |
| createdAt | LocalDateTime | 记录创建时间 |

**统计规则：** 按季度和历史总计自动累计每人各班次的天数。

### 2.3 授课记录（TeachingRecord）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Long (PK, 自增) | 主键 |
| staffId | Long (FK → Staff) | 关联的授课教练 |
| teachDate | LocalDate | 授课日期 |
| courseType | String | 上课类型（从预设列表选或手动输入） |
| studentCount | Int | 授课人数 |
| classHours | Double | 课时数 |
| createdAt | LocalDateTime | 记录创建时间 |

**统计规则：**
- 季度总课时 = 当前季度内该教练所有授课课时之和
- 历史总课时 = 全部授课课时之和
- 周期内入场总人数 = 所选时间范围内授课人数汇总

### 2.4 每周运营概况（WeeklyOperation）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Long (PK, 自增) | 主键 |
| weekStartDate | LocalDate | 周起始日期（周一） |
| totalVisitors | Int | 本周入馆总人数 |
| note | String | 备注（可选） |

### 2.5 预约记录（BookingRecord）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Long (PK, 自增) | 主键 |
| subject | String | 预约主体（预约方名称） |
| activityPlan | String | 活动方案 |
| bookingDate | LocalDate | 预约日期 |
| timeSlot | Enum | 时段：上午 / 下午 / 全天 |
| startTime | LocalTime | 详细开始时间（可微调） |
| endTime | LocalTime | 详细结束时间（可微调） |
| venueZoneId | Long (FK → VenueZoneConfig) | 功能区 |
| peopleCount | Int | 来访人数 |
| contactPerson | String | 对接人 |
| note | String | 备注（可选） |
| createdAt | LocalDateTime | 记录创建时间 |

**时间快捷选择规则：**
- 选择"上午" → 默认 08:00-12:00，可微调
- 选择"下午" → 默认 14:00-18:00，可微调
- 选择"全天" → 不显示详细时间编辑

### 2.6 场馆功能区配置（VenueZoneConfig）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Long (PK, 自增) | 主键 |
| name | String | 功能区名称（如：游泳区、篮球场） |
| sortOrder | Int | 排序号 |

### 2.7 上课类型配置（CourseTypeConfig）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Long (PK, 自增) | 主键 |
| name | String | 类型名称（如：私教课、团课、体验课） |
| sortOrder | Int | 排序号 |

### 实体关系图

```
Staff (1) ──〈 (N) DutyRecord
Staff (1) ──〈 (N) TeachingRecord
VenueZoneConfig (1) ──〈 (N) BookingRecord

WeeklyOperation (独立，按周记录)
CourseTypeConfig (独立，配置表)
```

## 三、功能模块

### 3.1 工作记录（首页）

- **默认视图：** 按日期倒序排列的值班+授课记录混合列表
- **快捷筛选：** 全部 / 今日 / 本周 / 本月 / 自定义日期范围
- **新增值班：** 选择日期 → 从人员列表点选 → 选择班次（早/晚/通大按钮）
  - 支持批量：一次选择多个日期 + 多个人员 + 统一班次
- **新增授课：** 选择日期 → 选择教练 → 选择上课类型（标签点选，可手动输入）→ 人数 → 课时
  - 支持连续添加模式

### 3.2 预约日历

- **月视图日历**：顶部月份切换，日期格子上显示预约数量小圆点
- **点击某天**：底部弹出当天预约列表，展示每条摘要
- **新增预约**：
  - 预约主体（输入框+记忆功能）
  - 活动方案（输入框）
  - 预约日期（日历点选）
  - 时段选择（上午/下午/全天三个大按钮）
  - 功能区（预设列表点选）
  - 人数（数字输入器）
  - 对接人（从工作人员列表点选）
  - 备注（输入框）

### 3.3 统计报表

三个子页面通过顶部 Tab 切换：

1. **值班统计**
   - 统计周期：本季度 / 上季度 / 本年度 / 自定义
   - 每人按班次分类展示：早班N次 | 晚班N次 | 通班N次 | 合计N次

2. **授课统计**
   - 统计周期：本季度 / 上季度 / 本年度 / 历史总计 / 自定义
   - 每人展示：总课时 / 总授课人数
   - 可展开查看明细

3. **运营概况**
   - 按周列表展示：周次区间 → 入馆总人数

### 3.4 管理设置

- **工作人员管理**：增删改人员，设置备注类型（值班/顶班），排序
- **功能区配置**：增删改功能区，排序
- **上课类型配置**：增删改默认类型，排序
- **数据导出**：选择范围 → Excel导出 / 文字模板导出
- **数据备份与恢复**：手动备份到云端 / 从云端恢复

## 四、交互与体验设计原则

1. **快速选择优先**：所有下拉菜单改为大按钮卡片或标签列表，减少点击次数
2. **批量操作**：值班支持多日多人的批量录入
3. **记忆功能**：预约主体等字段支持历史输入记忆
4. **连续录入**：授课录完后直接继续下一条
5. **可视化反馈**：日历上的小圆点标记、统计数据即时刷新
6. **自定义优先**：功能区、上课类型均可由管理员自由配置

## 五、数据流与存储

```
用户操作 → ViewModel → Repository → Room DAO → SQLite
                                      ↓
                               Auto Backup → Google Drive
```

### 备份策略
- **自动备份**：利用 Android Auto Backup 每天自动备份到 Google Drive
- **手动备份**：设置页提供"立即备份"按钮
- **手动恢复**：设置页提供"从备份恢复"按钮

### 导出
- **Excel 导出**：选择时间范围和数据类别（值班/授课/运营/全部），生成 .xlsx 文件
- **文字模板导出**：按固定模板生成文本总结报告，可分享或复制

## 六、非功能需求

- 最低支持 Android 8.0 (API 26)
- 离线可用，不需联网即可完整使用
- 少量数据占用，预估单条记录 < 1KB
- 支持中文字体显示
- 隐私说明：所有数据仅存储在用户设备和用户自己的云端账户

## 七、可扩展性设计

为确保系统易于后期维护和扩展，以下方面预留了扩展端口：

### 7.1 配置数据的统一接口

功能区配置、上课类型配置、工作人员管理采用相同的 **CRUD 架构模式**，各自拥有独立的 Repository + DAO 层，新增一种配置类型时只需：

```
1. 新建 Config 实体类（Room @Entity）
2. 新建 DAO 接口（继承通用 BaseDao 模式）
3. 新建 Repository（可复用通用增删改查逻辑）
4. 在设置页添加对应菜单入口
```

### 7.2 上课类型的弹性输入

- 既支持从预设配置列表中选取（数据驱动），也支持**手动输入新类型**
- 手动输入的值会自动记录，下次录入时可作为**快捷选项**出现（软预设）
- 未来可扩展为：自动统计手动输入频率，超过阈值提示管理员正式添加到预设列表

### 7.3 功能区配置的扩展点

- 功能区支持排序，未来可扩展为：
  - 功能区**分组**（如：水上项目区、球类区、健身区）
  - 功能区**可用时段**配置
  - 功能区**容量上限**配置（预约人数不可超过上限）

### 7.4 导出模板的扩展

- 文字模板导出采用**模板字符串 + 数据填充**模式
- 未来可扩展为：
  - 用户可自定义导出模板（在设置中编辑模板格式）
  - 支持更多导出格式（PDF、CSV）
  - 定时自动导出并分享

### 7.5 统计逻辑的扩展

- 统计模块采用**策略模式**，每种统计类型（值班/授课/运营）独立实现
- 未来可扩展：
  - 新增统计维度（如按月、按年同比）
  - 新增图表展示（柱状图、折线图）
  - 导出统计报告时包含图表

### 7.6 数据库迁移预留

- Room 数据库版本号已预留迁移路径
- 后期新增字段或表时，通过 Migration 对象平滑升级，不丢失已有数据
